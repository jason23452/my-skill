# 互動設計

本檔提供互動設計規則，適用於所有 command。

## 原則

- 每個互動都要有明確 affordance。
- hover、focus、active、disabled、loading、success、error 狀態都要設計。
- 觸控裝置不能依賴 hover。
- 鍵盤 focus order 應符合視覺順序。
- destructive action 需要 confirmation 或 undo。

## Feedback

- 快速動作提供即時回饋。
- 非同步操作顯示 loading 或 optimistic UI。
- 錯誤訊息放在發生位置附近。
- 成功訊息要說明結果，不只說 done。

## 完成標準

使用者能預測操作結果，並在錯誤或等待時知道發生什麼。
