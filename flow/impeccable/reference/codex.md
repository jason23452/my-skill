# Codex Visual Flow

本檔用於支援具有 native image generation 的 harness。在產生 mock 或視覺方向前必須先經過使用者確認。

## 流程

1. 確認 shape brief。
2. 問 1 到 3 個必要方向問題。
3. 確認 palette 與 visual references。
4. 產生多個 mock directions。
5. 讓使用者選擇、合併或授權 agent 決定。
6. 根據 approved direction 切出 assets 與實作 inventory。

## 禁止事項

- shape confirmed 不等於可以直接寫 code。
- 不可跳過 palette / mock approval gate。
- 不可把未確認 mock 當最終方向。

## 完成標準

進入實作前已有明確 approved visual direction。
