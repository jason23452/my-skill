# Colorize

`colorize` 用於替單調或缺乏品牌感的 UI 加入策略性色彩，而不是隨機加漸層。

## 流程

1. 先找既有 brand colors、tokens、theme。
2. 確認色彩策略：Restrained、Committed、Full palette、Drenched。
3. 定義角色：background、surface、ink、primary、accent、semantic。
4. 檢查 contrast。
5. 套用到 CTA、狀態、資料視覺、重點區塊。

## 規則

- 色彩要服務層級與任務，不是平均撒滿。
- 使用 OKLCH 時保持色階一致。
- semantic colors 不可和 brand accent 混淆。
- dark mode 不是 light mode 反相。

## 完成標準

色彩角色清楚、contrast 合格、品牌感提升，且不破壞可讀性。
