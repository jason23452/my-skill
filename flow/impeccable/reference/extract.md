# Extract

`extract` 用於從既有 UI 萃取 reusable components、tokens 與 design patterns。

## 流程

1. 找出 design system、component library、tokens 或 shared UI 位置。
2. 找重複三次以上或明確可重用的 pattern。
3. 分辨 token、component、composition pattern、utility class。
4. 設計 component API 與 migration path。
5. 不要過度抽象；只抽真正有價值的東西。

## 完成標準

減少重複與 drift，同時讓後續 UI 更一致、更容易維護。
