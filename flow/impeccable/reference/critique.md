# Critique

`critique` 用於設計層級審查，聚焦使用者能否理解、信任並完成任務。

## 評估面向

- Visual hierarchy
- Information architecture
- Emotional resonance
- Discoverability
- Composition
- Typography
- Color strategy
- State handling
- Microcopy
- Affordance

## 流程

1. 讀取目標畫面、程式碼或截圖。
2. 確認畫面要完成的主要任務。
3. 找出 AI slop、generic template、層級不清、內容過載或互動不明。
4. 將問題依影響排序。
5. 給出具體修正，不只說「更好看」。

## 完成標準

輸出可行的 design critique，指出哪些地方影響任務完成、理解、信任或品質感。
