# Onboard

`onboard` 用於設計 first-run experience、activation flow、empty states 與新手引導。

## 流程

1. 定義 activation moment：使用者第一次感受到價值的時刻。
2. 減少第一步摩擦，只要求必要資訊。
3. 使用 progressive disclosure。
4. Empty state 要包含 context、benefit、next action。
5. 避免冗長 tour；優先 in-context guidance。

## 完成標準

新使用者知道為什麼要繼續、下一步做什麼，且能快速到達價值點。
