# iOS 參考

原生 iOS UI 應遵守 Apple Human Interface Guidelines 的基本期待。

## 原則

- 尊重 safe area、Dynamic Type、Reduce Motion、VoiceOver。
- 使用 iOS familiar navigation：tab bar、navigation stack、sheet、context menu。
- Edge-swipe back 不可被破壞。
- 觸控目標要足夠大。

## 視覺

- 使用系統字體與平台慣例，除非品牌有明確理由。
- Dark mode 是獨立設計，不是反相。
- Motion 要符合 iOS 物理感，並支援 Reduce Motion。

## 完成標準

介面看起來像 iOS app，而不是把 web UI 包進 native shell。
