# Document

`document` 用於從既有專案 code 產生 DESIGN.md 或設計系統文件。

## 流程

1. 掃描 CSS、tokens、theme、components、pages。
2. 萃取色彩、typography、spacing、radius、shadow、motion、component patterns。
3. 記錄實際已存在的規則，不要發明不存在的設計系統。
4. 標出 inconsistency 與需要決策的 gap。
5. 產出可供後續 agent 遵守的文件。

## 完成標準

文件準確反映現有 UI，能作為後續設計與實作的 source of truth。
