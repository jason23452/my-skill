# 塑形

`shape` 用於寫 code 前塑形 UX/UI。目標是產生可交給實作的設計 brief，而不是直接跳到卡片網格或安全模板。

## 流程

1. 讀取產品脈絡、需求、目標使用者、主要任務與限制。
2. 找出使用者真正要完成的 primary action。
3. 定義 layout strategy、資訊層級、主要狀態、互動模式與內容需求。
4. 若方向不明，提出 2 到 3 個設計方向與 trade-offs。
5. 在進入實作前停下來讓使用者確認或修正。

## 輸出格式

- Feature summary
- Primary user action
- 設計方向
- 版面策略
- Key states
- Interaction model
- Content requirements
- 參考建議
- Open questions

## 完成標準

設計 brief 足以讓另一個 agent 不猜測也能實作 UI，且不會落入 generic SaaS / dashboard 模板。
