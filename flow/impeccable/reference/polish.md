# 打磨

`polish` 是上線前最後品質 pass，用來修正那些讓 UI 從可用變成精緻的細節。

## 檢查項目

- Alignment、spacing、rhythm、visual balance。
- Typography scale、line-height、line length、wrapping。
- Color contrast、token consistency、dark mode。
- Hover、focus、active、disabled、loading、error、empty states。
- Responsive breakpoints、touch targets、overflow。
- Motion timing、easing、reduced motion。
- Copy clarity、button labels、error messages。

## 流程

1. 先理解既有 design system。
2. 找出 drift：hard-coded values、重複元件、不一致 spacing。
3. 修正高影響細節。
4. 驗證主要 viewport 與互動狀態。

## 完成標準

沒有明顯粗糙邊角，UI 在主要情境下可讀、可操作、狀態完整且視覺一致。
