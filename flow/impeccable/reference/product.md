# Product Register

當 design 服務產品任務時使用：app UI、admin、dashboard、tools、settings、forms、workflows。

## 原則

- 主要任務優先於視覺表演。
- 層級、密度、狀態與 feedback 要清楚。
- 保留品牌感，但不要犧牲可讀性與效率。
- 資料密集畫面需要清楚 grouping、filters、sorting、empty/error/loading states。

## 版面

- App shell 應穩定，navigation 不要讓使用者迷路。
- Primary action 要明確。
- Secondary actions 不要和 primary CTA 搶權重。
- 表格、圖表、表單要針對 mobile 與 keyboard 使用者設計。

## 完成標準

使用者能快速理解目前位置、資料狀態與下一步操作。
