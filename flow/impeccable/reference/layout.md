# 版面

`layout` 用於修復 spacing、rhythm、composition、visual hierarchy 與 responsive 結構。

## 流程

1. 找出目前 layout 問題：擠、散、失衡、過多 card、層級不明、手機跑版。
2. 重新定義內容優先級與視覺流向。
3. 選擇正確結構：single column、split layout、sidebar、grid、stack、cluster。
4. 用 spacing scale 建立節奏，不要每段都同樣 padding。
5. 驗證 mobile、tablet、desktop。

## 規則

- Cards 只在真的需要容器時使用。
- Grid 用於二維對齊，Flexbox 用於一維排列。
- 重要內容不要被 decoration 或 sidebar 擠壓。
- 版面要有呼吸感，但不可犧牲任務完成。

## 完成標準

使用者能自然知道先看哪裡、下一步做什麼，且不同螢幕尺寸都不破版。
