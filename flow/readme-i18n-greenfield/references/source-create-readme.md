# 來源 Skill：create-readme

來源：`github/awesome-copilot/create-readme`

## 中文改寫摘要

`create-readme` 的核心目標是建立專業、資訊清楚、容易閱讀的 `README.md`。整合到本 skill 時，採用以下規則：

- 先完整檢查 project / workspace，再撰寫 README。
- README 應該有清楚結構、專業語氣與可掃讀的段落。
- 內容要精簡到位，不要把 README 寫成流水帳。
- 不要濫用 emoji。
- 不要加入 `LICENSE`、`CONTRIBUTING`、`CHANGELOG` 這類通常應由獨立檔案承載的段落。
- 使用 GitHub Flavored Markdown。
- 如果 repo 有 logo、icon 或品牌圖片，才可放在 README header；不要自行發明。

## 套用到本專案的調整

本專案的 Greenfield README 預設使用繁體中文，因此 `create-readme` 的英文輸出風格被改寫為中文專業文件風格。README 第一屏應該先說明專案用途，而不是寫 `Generated by OpenCode` 或 bootstrap evidence。
